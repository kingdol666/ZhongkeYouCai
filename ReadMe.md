# CSV 数据批处理工具 — 使用说明

## 1. 软件概述

本软件用于模头厚度测量数据的自动采集、合并、热力云图生成与存档。支持对生产线文件夹的实时监控，检测新增 CSV 文件后自动触发处理流程，并可在每日零点自动切换监控日期文件夹。

软件采用 PyQt6 多线程架构，界面包含五个功能标签页。

---

## 2. 标签页功能详解

### 2.1 数据处理（批量模式）

遍历输入文件夹下的**所有子文件夹**，对每个子文件夹合并其 CSV 文件并输出。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 输入文件夹 | 包含多个子文件夹的目录 | — |
| 输出文件夹 | 合并后 CSV 的保存目录 | `ProcessedData/` |
| 起始行号 | 每个 CSV 从第几行开始读取 | 8 |
| 终止行号 | 每个 CSV 读取到第几行 | 190 |
| 起始模头位置 | 第一列对应的物理位置 (mm) | 2.50 |
| 终止模头位置 | 最后一列对应的物理位置 (mm) | 14.39 |

**操作**：设置参数 → 点击"开始处理" → 完成后在输出目录查看 `merged_*.csv`。

---

### 2.2 单文件夹分析

对**单个子文件夹**执行合并 + 自动生成热力云图，两步合并为一个操作。

- 选择目标文件夹（如 `Test_xxx/定量/`）
- 输出：合并 CSV + 热力云图 PNG
- 云图实时显示在界面中，双击可切换原始大小/缩放

---

### 2.3 文件夹监控（核心功能）

实时监控指定文件夹，检测新增 CSV 文件后自动触发"合并 + 云图"处理。

#### 监控逻辑

| 场景 | 行为 |
|------|------|
| CSV 出现在**与上次相同**的父目录 | 直接处理，**不**存档 |
| CSV 出现在**与上次不同**的父目录 | **先自动存档**上一批结果，再处理新的 |
| 首次检测到 CSV | 视为"不同父目录" |

父目录 = CSV 文件所在的 `定量/` 文件夹的**绝对路径**。

#### 界面参数

| 参数 | 说明 |
|------|------|
| 监控文件夹 | 实际监控的路径（自动切换模式下由程序自动填充） |
| 忽略文件夹 | 跳过监视的文件夹名，逗号分隔，默认 `定量_Roll, 定量_RollTrend` |
| 临时输出 | 合并 CSV 和云图的临时存放目录 |
| 存档文件夹 | 换卷时自动存档的目标目录（带时间戳子文件夹） |
| CustomProfile | 日期文件夹的根目录（启用自动切换后必填） |
| 每日零点自动切换 | 勾选后启用 |

#### 每日零点自动切换

勾选后，程序每 30 秒检查一次当前日期：

1. 启动监控时，自动拼接 `{CustomProfile}/{year}_{month}/{day}` 作为真实监控路径
2. 目标文件夹不存在则自动创建
3. 零点跨天后，**停止旧日期文件夹的监控**，**启动新日期文件夹的监控**
4. 同时扫描磁盘上是否有更新的日期文件夹（匹配 `YYYY_MM/DD` 结构），有则自动跟随

**目录结构示例**：
```
CustomProfile/
├── 2026_05/
│   ├── 07/
│   │   ├── Test_Tes..._141_0/
│   │   │   ├── 定量/            ← 被监控
│   │   │   ├── 定量_Roll/       ← 忽略
│   │   │   └── 定量_RollTrend/  ← 忽略
│   │   └── Test_Tes..._142_0/   ← 换卷后
│   │       └── 定量/
│   └── 08/                      ← 零点后切换至此
│       └── Test_Tes..._143_0/
│           └── 定量/
```

#### 按钮说明

| 按钮 | 功能 |
|------|------|
| 开始监控 | 启动文件夹监控（含零点定时器） |
| 停止监控 | 停止监控和定时器 |
| 手动存档 | 将当前显示的 CSV 和云图手动保存到存档目录 |
| 模拟零点切换 | 立即触发零点切换逻辑（测试用），优先使用磁盘上最新日期文件夹 |

---

### 2.4 热力云图

对已合并的 CSV 文件单独生成热力云图。

- 选择 `merged_*.csv` 文件
- 点击"生成热力图"
- 输出 300 DPI 的 PNG 图像

## 3. 产线仿真客户端

`tools/simulate_production.py` 用于验证监控全流程，模拟真实产线行为。

### 启动方式
```bash
python tools/simulate_production.py
```

### 目录结构（自动创建）
```
sim_test/                        ← 仿真根目录（默认）
├── 2026_05/
│   ├── 07/                      ← 仿真日期
│   │   ├── Test_..._141_0/      ← 卷膜
│   │   │   ├── 定量/
│   │   │   ├── 定量_Roll/
│   │   │   └── 定量_RollTrend/
│   │   └── Test_..._142_0/      ← 换卷后
│   └── 08/                      ← "过了一天"后
```

### 操作按钮

| 按钮 | 功能 |
|------|------|
| 1. 设置模拟环境 | 初始化仿真目录结构，加载 CSV 源队列 |
| 2. 生产CSV | 从源队列复制一个 CSV 到当前定量文件夹 |
| 3. 换卷 | 当前日期下创建新 Test_* 卷膜目录 |
| 4. 过了一天 | 仿真日期 +1，创建新日期文件夹，重置卷膜计数 |
| 自动生产(2s) | 每 2 秒自动生产一个 CSV |
| 启动监控 | 内置 MonitorCore 监控仿真目录 |
| 停止监控 | 停止内置监控 |

### 与主应用联调

1. 仿真客户端：设置环境，开始生产
2. 主应用：CustomProfile 设为 `sim_test/`，勾选自动切换，启动监控
3. 仿真客户端点击**过了一天** → 主应用 30 秒内自动检测新日期文件夹并切换
4. 或主应用点击**模拟零点切换**立即跟随

## 4. 数据格式要求

### CSV 文件命名
`时_分_秒_序号.CSV`，如 `21_51_29_1.CSV`

### CSV 文件内容
两列数据，无表头：
- 第一列：位置编号
- 第二列：厚度值 (bw)

### 文件编码
支持 UTF-8、GBK，自动检测（chardet 兜底）。

## 6. 常见问题

**Q: 监控启动后没反应？**
确认监控路径下存在包含 CSV 文件的子目录。使用仿真客户端可快速搭建测试环境。

**Q: 零点切换不生效？**
确认已勾选"每日零点自动切换"且 CustomProfile 路径正确。程序每 30 秒检查一次，切换后日志会有记录。

**Q: 自动存档位置？**
存档到"存档文件夹"下以 `result_YYYYMMDD_HHMMSS` 命名的子文件夹中。

**Q: CSV 读取失败？**
程序依次尝试 UTF-8 → GBK → chardet 自动检测，确保文件未损坏。
